## 第一章 核心基础规范
### 1.1 节点块核心规则
所有主线节点、角色事件、共同事件，必须用`---`包裹形成**独立完整的元数据块**，元数据块内仅可放置字段，不可混入正文、注释等内容。
```markdown
---
ID: V1M01
名称：乱葬岗醒魂
视角：男主
地图：青阳城·乱葬岗
前置：无
解锁周目：1
补触发：无
---
```

### 1.2 字段格式统一规范
| 字段类型 | 固定格式 | 示例 | 代码识别规则 |
|----------|----------|------|--------------|
| 单行文本 | 字段名：值 | 名称：乱葬岗醒魂 | 正则匹配`^[\u4e00-\u9fa5]+：.*$` |
| 空值 | 字段名：无 | 前置：无 | 空值必须填「无」，不可留空、不可填「-」「/」 |
| 多值 | 字段名：值1 + 值2 | 前置：V1M10 + V1F04 | 多值之间用「 + 」（空格+加号+空格）分隔，不可用其他符号 |
| 数字 | 字段名：数字 | 解锁周目：1 | 仅可填写阿拉伯数字，不可带单位、符号 |

### 1.3 元数据块必填字段规范
所有主线节点、共同事件的元数据块，必须包含以下7个必填字段，顺序固定不可调换，不可遗漏、不可新增非标必填字段：
| 字段名 | 格式约束 | 枚举/示例 | 说明 |
|--------|----------|-----------|------|
| ID | 固定格式：V{卷号}{视角缩写}{序号} | V1M01、V1F04、V1C01 | 视角缩写：M=男主、F=女主、C=共同；序号为2位阿拉伯数字，不足补0 |
| 名称 | 单行文本 | 乱葬岗醒魂 | 节点/事件的官方名称，不可超过20字 |
| 视角 | 固定枚举值 | 男主 / 女主 / 共同 | 仅可填写三个枚举值中的一个，不可用英文、其他表述 |
| 地图 | 单行文本 | 青阳城·乱葬岗 | 格式：大地图·子场景，无则填「无」 |
| 前置 | 空值/单ID/多值 | 无 / V1M04 / V1M10 + V1F04 | 节点触发的前置条件，多值用「 + 」分隔 |
| 解锁周目 | 数字 | 1 | 触发所需最低周目数，无门槛填1 |
| 补触发 | 空值/节点ID | 无 / V2M03 | 错过节点后的保底补触发节点ID，无则填「无」 |

## 第二章 主线节点内容区块规范
元数据块下方为内容区块，严格遵循以下必填/可选规则、格式规范，区块顺序固定不可调换。

### 2.1 内容区块必填/可选规则
| 区块名 | 必填性 | 固定格式 | 说明 |
|--------|--------|----------|------|
| 【正文】 | ✓ 强制必填 | 【正文】+ 换行 + 剧情文本 | 节点核心剧情内容，不可为空 |
| 【NPC】 | ○ 可选 | 【NPC】+ 换行 + 角色名：对话内容 | NPC对话内容，每行一条，不可嵌套 |
| 【选择】 | ✓ 强制必填 | 【选择】+ 换行 + 有序列表选项 | 节点跳转选项，至少1条，不可为空 |
| 【效果】 | ○ 可选 | 【效果】+ 换行 + 效果列表 | 节点触发后的全局效果，所有选项共享 |
| 【补触发规则】 | ○ 可选 | 【补触发规则】+ 换行 + 说明文本 | 补触发的详细规则说明，代码不解析 |
| 【多周目规则】 | ○ 可选 | 【多周目规则】+ 换行 + 规则列表 | 多周目特殊触发规则，代码不解析 |
| 【全局伏笔】 | ○ 可选 | 【全局伏笔】+ 换行 + 伏笔说明 | 世界观伏笔回收说明，代码不解析 |
| 【共同事件触发】 | ○ 可选 | 【共同事件触发】+ 事件链接 | 触发的共同事件引用，代码不解析 |

### 2.2 选项格式强制规范
【选择】区块必须使用有序列表，严格遵循以下格式，不可自定义格式：
```markdown
【选择】
1. {选项文本} → {目标节点ID}
2. {选项文本} → {目标节点ID}
   效果：{选项特有效果描述}
```
- 选项与目标节点ID之间用「 → 」（空格+箭头+空格）分隔，不可用其他符号
- 选项特有效果必须换行缩进，以「效果：」开头，仅对应当前选项
- 目标节点ID必须为已存在的有效节点ID，不可为空

### 2.3 主线节点完整合规示例
```markdown
---
ID: V1M05
名称：城中异闻
视角：男主
地图：青阳城·街道
前置：V1M04
解锁周目：1
补触发：V2M03
---

【正文】
你进入青阳城。
街道上行人匆匆，却有不少修士面色呆滞，眼神空洞，如同失了魂。

【NPC】
路人修士：唉，最近又有好几个人失了记忆，真是诡异……
路人修士：听说圣女大人下山了，只有她能镇压此地邪祟。

【选择】
1. 上前询问修士详情 → V1M07
2. 不理会，独自前往古井探查 → V1M08

【效果】
得知：近期大量修士失忆
解锁线索：圣女苏清鸢在城中

【补触发规则】
卷二回城节点V2M03保底触发本节点未触发的支线

【全局伏笔】
凡人失魂是天道规则崩坏的前兆，卷五回收
```

## 第三章 角色支线事件规范
### 3.1 角色文件头部规范
每个角色单独一个MD文件，文件命名固定为`char-3位数字.md`（例：`char-001.md`），文件头部必须包含角色信息头，格式固定如下：
```markdown
# 角色：{角色名}
角色ID：C{3位数字}

| 基础信息 | 内容 |
|----------|------|
| 姓名 | {角色姓名} |
| 性别 | {男/女} |
| 身份 | {角色核心身份描述} |
| 关联主线 | {对应主线节点ID} |
| 解锁周目 | {数字} |
| 补触发节点 | {节点ID/无} |
```
- 角色ID固定格式：`C+3位数字`，不足补0，不可重复、不可跳号
- 核心角色固定ID：凌辰=C000、苏清鸢=C001、林清寒=C002、阿翠=C003、墨老=C004、小豆子=C005

### 3.2 角色事件元数据块规范
角色事件必须用`---`包裹元数据块，必填字段、格式固定如下，顺序不可调换：
```markdown
---
事件ID：C{角色3位数字}_E{2位序号}
事件名：{事件名称}
触发类型：{固定枚举值}
优先级：{数字}
解锁周目：{数字}
前置条件：
  {条件1}
  {条件2}
补触发：{节点ID/无}
---
```
- 事件ID固定格式：`C角色ID_E序号`，例：`C001_E01`
- 优先级：数字越小，触发优先级越高，同节点内按优先级排序
- 前置条件：支持多行换行、组合条件，格式遵循第四章规范

### 3.3 触发类型枚举与配置规范
| 触发类型 | 说明 | 额外必填字段 | 示例 |
|----------|------|--------------|------|
| 强制 | 100%必触发 | 无 | 触发类型：强制 |
| 概率 | 概率触发，支持周目递增保底 | 基础概率、周目递增 | 基础概率：50%<br>周目递增：+30%/周目 |
| 周目 | 达到指定周目自动触发 | 无 | 触发类型：周目 |
| 好感 | 角色好感度达标触发 | 好感门槛 | 好感门槛：苏清鸢好感 ≥ 50 |
| 物品 | 持有指定物品触发 | 无 | 触发类型：物品 |

## 第四章 前置条件表达式规范
所有前置条件必须严格遵循以下格式，支持单条件、多条件组合、分支条件，代码可直接解析。

### 4.1 单条件类型与固定格式
| 条件类型 | 固定格式 | 合规示例 |
|----------|----------|----------|
| 周目大于等于 | 周目 ≥ N | 周目 ≥ 3 |
| 周目等于 | 周目 = N | 周目 = 2 |
| 周目小于等于 | 周目 ≤ N | 周目 ≤ 5 |
| 好感度大于等于 | {角色名}好感 ≥ N | 苏清鸢好感 ≥ 50 |
| 好感度小于 | {角色名}好感 < N | 林清寒好感 < 30 |
| 节点完成 | 已完成 {节点ID} | 已完成 V1M08 |
| 事件触发 | 已触发 {事件ID} | 已触发 C001_E01 |
| 持有物品 | 持有 {物品名} | 持有 墨铜钱 |
| 物品数量 | 持有 {物品名} ≥ N | 持有 轮回碎玉 ≥ 9 |

| 选项分支条件 | {事件ID}选择{选项序号} | C001_E01选择2 |

### 4.2 组合条件格式规范
| 组合类型 | 格式 | 合规示例 |
|----------|------|----------|
| AND组合（同时满足） | 多行换行 / 「且」连接 | 前置条件：<br>  周目 ≥ 3<br>  苏清鸢好感 ≥ 50<br>  已完成 V1M08 |

| 复杂嵌套 | 括号区分优先级 | 前置条件：<br>  周目 ≥ 2<br>  （当前路线: 正道 且 林清寒好感 ≥ 30）或 持有 推荐信 |
